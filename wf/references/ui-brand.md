# WF 视觉规范

## 阶段横幅

每个主要工作流步骤开始时显示横幅：

```
╔══════════════════════════════════════════╗
║  WF · {{阶段名称}}                       ║
╚══════════════════════════════════════════╝
```

## 检查点框

流程中的检查点使用方框显示：

```
┌─ 检查点 ─────────────────────────────────┐
│ ✅ 需求文档已生成                         │
│ ✅ 路线图已确认                           │
│ ⬜ 等待用户确认                           │
└───────────────────────────────────────────┘
```

## 状态符号

| 符号 | 含义 |
|------|------|
| ✅ | 完成 |
| ⬜ | 待处理 |
| 🔄 | 进行中 |
| ❌ | 失败 |
| ⚠️ | 警告 |
| ⛔ | 阻塞 |
| ▶ | 下一步 |

> **注意:** 上述状态符号是 WF 系统的视觉设计选择，用于在终端中提供清晰的状态区分，不受通用 no-emoji 指导的约束。

## 进度显示

```
阶段 3/5 ████████░░░░░░░░ 47%
```

进度条规则：
- 使用 `█` 表示已完成部分
- 使用 `░` 表示未完成部分
- 总长度 16 字符
- 右侧显示百分比

## 路由显示

当自动路由到下一步时：

```
▶ 下一步: /wf-plan-phase 2
  原因: 阶段 2 已完成讨论，准备生成计划
```

## 错误显示

```
⛔ 阻塞: {{原因}}
   {{建议操作}}
```

## Agent 启动显示

```
🚀 启动 wf-executor [阶段 3, 计划 A]
   隔离模式: worktree | 分支: wf/phase-3-plan-a
```
