<objective>
Create the /wf-settings command for interactive configuration management with both menu-based and direct CLI modes.

Purpose: Users currently must manually edit `.planning/config.json` to change workflow behavior. This plan creates a user-friendly settings command that exposes behavior configs (per D-08) through both interactive AskUserQuestion menus (per D-07) and direct `set key value` CLI operations.

Output: New `commands/wf/settings.md`, new `wf/workflows/settings.md`, extended `config.cjs` with save/schema capabilities, updated `wf-tools.cjs` router.

Note: This plan is Wave 2 because it modifies `wf-tools.cjs` which is also modified by Plan 02 (phase-ops). Plan 02 must complete first to avoid file conflicts.
</objective>

**Add `saveConfig(cwd, key, value)` function to `config.cjs`:**

Implement a function that:
1. Reads existing `.planning/config.json` via `utils.readJson(configPath) || {}`
2. Supports dotted keys ("workflow.research") by splitting on `.` and navigating/creating nested objects
3. Calls `parseConfigValue(value)` to coerce strings to proper types before saving
4. Writes back with `utils.writeFile(configPath, JSON.stringify(existing, null, 2))`
5. Returns `{ success: true, key, value: parsedValue }`

**Add `parseConfigValue(value)` helper** (not exported):
- 'true' -> true (boolean)
- 'false' -> false (boolean)
- /^\d+$/ match -> parseInt(value, 10)
- /^\d+\.\d+$/ match -> parseFloat(value)
- Otherwise -> string as-is

**Add `getConfigSchema()` function:**
Recursively flatten CONFIG_DEFAULTS into an array of `{ key, type, default }` objects. Keys starting with "_" are excluded (per D-08). Nested objects are flattened with dot notation (e.g., "workflow.research").

**Add `getConfigValue(cwd, key)` function:**
Load merged config via `loadConfig(cwd)`, navigate dotted key path, return `{ key, value, is_default }`.

**Update `run(cwd, args)` function** to handle sub-commands:
- No args: keep existing behavior (output full merged config)
- `['set', key, value]`: call saveConfig, output result
- `['get', key]`: call getConfigValue, output result
- `['schema']`: call getConfigSchema, output result

Change signature from `run(cwd)` to `run(cwd, args)` with `const sub = (args || [])[0]` guard.

**Update `module.exports`**: add saveConfig, getConfigSchema, getConfigValue, deepMerge to exports.

**Update wf-tools.cjs**: Change `config.run(cwd)` to `config.run(cwd, subArgs)`. Add `case 'settings': config.run(cwd, subArgs); break;` as alias. Update help text to include "settings".

**Extend `config.test.cjs`** with tests for saveConfig, getConfigSchema, getConfigValue, and the new run sub-commands. Follow existing test pattern with tmp directories.

**Create `wf/workflows/settings.md`** (per D-09):

Structure with 3 steps:

**Step 1: Detect Mode**
- Check if $ARGUMENTS contains "set" keyword
- If yes: direct CLI mode (Step 3)
- If no args: interactive menu mode (Step 2)

**Step 2: Interactive Menu Mode (per D-07)**
- Run `node "$HOME/.claude/wf/bin/wf-tools.cjs" settings schema` to get all configurable keys
- Run `node "$HOME/.claude/wf/bin/wf-tools.cjs" settings` to get current merged config
- Group config keys by category for display:
  - 基本设置: mode, granularity
  - 工作流行为: workflow.research, workflow.plan_check, workflow.verifier, workflow.auto_advance, workflow.security_enforcement, workflow.discuss_mode
  - 并行化: parallelization.enabled, parallelization.plan_level, parallelization.max_concurrent_agents
  - 门禁: gates.confirm_project, gates.confirm_phases, gates.confirm_roadmap, gates.confirm_plan
  - 安全: safety.always_confirm_destructive, safety.always_confirm_external_services
  - Agent 模型: agents.models.executor, agents.models.planner, agents.models.verifier, agents.models.researcher
- Display current values in a formatted table
- Use AskUserQuestion to ask which setting to modify (or "done" to exit)
- When user selects a key, show current value and ask for new value via AskUserQuestion
- Run `node "$HOME/.claude/wf/bin/wf-tools.cjs" settings set <key> <value>` to save
- Loop back to menu until user says "done"

**Step 3: Direct CLI Mode (per D-07)**
- Parse "set key value" from $ARGUMENTS
- Run `node "$HOME/.claude/wf/bin/wf-tools.cjs" settings set <key> <value>`
- Display result

**Key constraints:**
- Only expose keys from CONFIG_DEFAULTS structure (per D-08)
- Keys starting with _ are never shown (per D-08)
- All writes go to .planning/config.json only (per Pitfall 3 in RESEARCH.md)
- Unknown keys in existing config.json are preserved but not displayed
- Keep file under 120 lines
