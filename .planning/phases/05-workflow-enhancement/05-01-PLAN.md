**Structure (6 steps):**

**Step 1: Parse Flags and Discover Phases**
- Parse flags from $ARGUMENTS: `--from N`, `--to N`, `--only N`, `--interactive`
- Run `node "$HOME/.claude/wf/bin/wf-tools.cjs" roadmap analyze` to get all phases
- Determine execution range:
  - Default: from first non-verified phase to last phase
  - `--from N`: start at phase N
  - `--to N`: end at phase N (inclusive)
  - `--only N`: execute only phase N
- Display execution banner per ui-brand.md style showing phase range

**Step 2: Phase Loop (for each pending phase N)**

Sub-step 2.1 - Context Budget Check:
- Before each phase, run `node "$HOME/.claude/wf/bin/wf-tools.cjs" state json` and check if context budget concern exists
- If wf-context-monitor has recorded remaining < 40%, pause autonomous:
  ```bash
  node "$HOME/.claude/wf/bin/wf-tools.cjs" session pause --phase <N> --plan 0 --step discuss --stopped_at "Context budget low, pausing autonomous"
  ```
  Display message: "Context budget low. Run `/wf-autonomous --from N` in a new session."
  Stop loop.

Sub-step 2.2 - Discuss Phase (per D-01, D-03):
- Default mode: `Skill(discuss-phase, { phase: N, flags: "--auto --batch" })`
- If `--interactive` flag was passed: `Skill(discuss-phase, { phase: N })`
- No confirmation needed between discuss and plan.

Sub-step 2.3 - Plan Phase (per D-01):
- `Skill(plan-phase, { phase: N })`
- Plan quality gate handled internally by plan-phase workflow (up to 3 revisions).
- No confirmation needed.

Sub-step 2.4 - Execute Phase (per D-01):
- `Skill(execute-phase, { phase: N })`

Sub-step 2.5 - Verify and Gap Closure (per D-02):
- `Skill(verify-work)`
- Check verification result. If FAIL:
  1. Display "Verification failed for Phase N, attempting gap closure..."
  2. Generate and execute gap closure plan (one retry)
  3. `Skill(verify-work)` again
  4. If still FAIL: pause autonomous, display failure details, suggest `/wf-autonomous --from N`
  5. Stop loop. Do NOT skip to next phase.

Sub-step 2.6 - Advance to Next Phase:
- Display completion banner: "Phase N complete"
- Run `node "$HOME/.claude/wf/bin/wf-tools.cjs" state begin-phase --phase <N+1>` (only if there is a next phase)

**Step 3: Completion**
- After all phases complete, display summary showing each phase result
- Suggest `/wf-verify-work` for final acceptance

**Error Handling Section:**
- Single task failure: record in SUMMARY.md, continue other tasks in the plan
- Entire plan failure: pause, display error, wait for user
- Verification failure: single gap closure attempt, then pause (per D-02)
- Context exhaustion: save state via session pause CLI, prompt user for new session

**Key constraints in the file:**
- Use `Skill()` for all workflow invocations, not `Task()` (per Phase 4 convention)
- All state updates through `wf-tools state` CLI commands, never direct Write/Edit
- Route targets limited to known Skill names (discuss-phase, plan-phase, execute-phase, verify-work)
- Keep file under 200 lines (current pseudocode is 161, executable version should be similar)
