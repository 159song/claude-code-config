   - Read ROADMAP.md content
   - Run `roadmapAnalyze(cwd)` to find all existing phases
   - Calculate next integer: `Math.max(...phases.map(p => Math.floor(p.num))) + 1`
   - Build new phase block (### Phase heading, **Goal**, **Depends on**: TBD, **Requirements**: TBD, **Plans**: TBD)
   - Find the `## Progress` section in ROADMAP.md (or end of file if absent)

   - Return `{ success: true, phase_number: phaseNum, archived_to: archivePath }`

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | ROADMAP corruption | mitigate | Re-validate with roadmapAnalyze() after every write; if parse result is unexpected, rollback (re-read and abort) |
| T-05-05 | Tampering | Path traversal via phase name | mitigate | Phase names used only in ROADMAP.md text, not in filesystem paths; directory names come from existing phase directories only |
| T-05-06 | Tampering | removePhase archive path | mitigate | Archive destination uses basename of existing directory (no user-provided path components in fs.renameSync target) |
| T-05-07 | Information Disclosure | removePhase preserves data | accept | Archived directories remain in .planning/archive/ by design (D-05); user controls git history |
</threat_model>
