<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> version string | Version used in filesystem paths and git tags |
| Archive operation -> filesystem | Recursive directory copy could be expensive |
| State reset -> filesystem | Destructive operation removing phase directories |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-11 | Tampering | complete-milestone version input | mitigate | Validate version matches /^v\d+\.\d+$/ regex in milestone.cjs archiveMilestone (already implemented in Plan 01 T-06-05) |
| T-06-12 | Denial of Service | complete-milestone archive large phases | accept | Archive is a one-time operation per milestone; large phases are expected; no timeout concern for local filesystem |
| T-06-13 | Tampering | resetForNewMilestone deleting files | mitigate | Reset only operates within .planning/phases/ and specific files; preserves PROJECT.md, STATE.md, config.json, milestones/; archive MUST complete before reset runs |
| T-06-14 | Repudiation | Milestone completion without git tag | mitigate | complete-milestone workflow creates git tag before reset; tag provides immutable history marker |
</threat_model>
