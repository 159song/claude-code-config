<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> file system | --files flag paths could traverse outside project root |
| CLI args -> git commands | Phase number injected into git diff commands |
| REVIEW.md -> consumer | Malformed YAML frontmatter could cause parse errors |
| Milestone version -> filesystem | Version string used in directory creation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | review.cjs computeFileScope | mitigate | Validate all --files paths resolve within cwd using path.resolve + startsWith check; reject traversal attempts |
| T-06-02 | Tampering | review.cjs getGitDiffFiles | mitigate | Use execFileSync (not exec) to prevent shell injection; paddedPhase validated as numeric in init.cjs |
| T-06-03 | Information Disclosure | review.cjs filterReviewFiles | mitigate | Exclude .env*, credentials, secrets files from review scope via filterReviewFiles allowlist |
| T-06-04 | Tampering | review.cjs parseReviewFrontmatter | mitigate | Validate parsed status is one of ['clean', 'issues_found', 'error']; reject unexpected values |
| T-06-05 | Tampering | milestone.cjs archiveMilestone | mitigate | Validate version matches /^v\d+\.\d+$/ regex; reject arbitrary strings that could create unexpected paths |
| T-06-06 | Denial of Service | milestone.cjs resetForNewMilestone | accept | Reset only operates within .planning/ directory; accidental deletion limited to planning artifacts which are git-tracked |
</threat_model>
