```javascript
'use strict';

// lib/validate.cjs -- WF 状态文件健康检查和格式验证模块
// 检测 STATE.md 格式问题并支持自动修复

const path = require('path');
const utils = require('./utils.cjs');
```

1. `validateHealth(cwd, repair)`:
   - Read `.planning/STATE.md` via utils.readFile. If null, return `{ valid: false, issues: ['STATE.md not found'], repaired: [] }`
   - Initialize `issues = []` and `repaired = []`
   - **Rule 1 - Frontmatter opener**: Check `content.startsWith('---\n')`. If false, push `'missing frontmatter opener'` to issues. If repair: prepend `---\nstatus: unknown\nlast_updated: "${new Date().toISOString()}"\nlast_activity: ${new Date().toISOString().slice(0,10)}\n---\n` to content, push to repaired.
   - **Rule 2 - Frontmatter closer**: Check `content.indexOf('\n---\n', 4) !== -1` (only if opener exists or was repaired). If false, push `'missing frontmatter closer'`. If repair: find last line matching `/^\w[\w_]*:/` pattern, insert `\n---\n` after it, push to repaired.
   - **Rule 3 - Required keys**: After opener+closer are confirmed/repaired, extract YAML block between `---` markers. Check for presence of `status:`, `last_updated:`, `last_activity:` using regex `/^status:\s/m`, `/^last_updated:\s/m`, `/^last_activity:\s/m`. For each missing: push `'missing required key: <keyname>'`. If repair: append the missing key line before the closing `---` with defaults:
     - `status: unknown`
     - `last_updated: "${new Date().toISOString()}"`
     - `last_activity: ${new Date().toISOString().slice(0,10)}`
     Push to repaired.
   - If repair and repaired.length > 0: write content back via utils.writeFile
   - Return `{ valid: issues.length === 0, issues, repaired }`

2. `validateFormat(cwd)`:
   - Read STATE.md. If no frontmatter, return `{ valid: false, issues: ['no frontmatter found'] }`
   - Extract YAML block
   - **Duplicate key check**: Parse top-level keys, if any key appears more than once, push `'duplicate key: <keyname>'`
   - Return `{ valid: issues.length === 0, issues }`

3. `run(cwd, args)`:
   - `const sub = args[0]`
   - If `sub === 'health'`: check for `--repair` in remaining args (`args.includes('--repair')`), call `validateHealth(cwd, hasRepair)`, output result
   - If `sub === 'format'`: call `validateFormat(cwd)`, output result
   - Else: `utils.error('用法: wf-tools validate [health|format] [--repair]')` and `process.exit(1)`

4. `module.exports = { validateHealth, validateFormat, run };`

Run tests, confirm all pass.
