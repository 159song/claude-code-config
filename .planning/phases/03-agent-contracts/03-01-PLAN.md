<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent output -> Workflow parser | Agent outputs text that workflow must parse for JSON completion marker; untrusted text could contain malformed JSON |
| Config.json -> Agent invocation | Agent model values read from config could be invalid model names |
| /tmp/ bridge file -> Executor | Context metrics read from /tmp/ could be stale, missing, or malformed |
| Agent artifacts array -> Filesystem | Artifact paths in completion markers could reference files outside project scope |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Completion marker JSON parsing | mitigate | Workflow must defensively parse: extract last JSON code block, validate status is one of "complete"/"partial"/"failed", default to "failed" if unparseable. Documented in agent-contracts.md |
| T-03-02 | Tampering | config.json agents.models values | mitigate | Workflow should validate model value against allowlist ["sonnet", "opus", "haiku", "inherit"] before passing to Agent(). Documented in agent-contracts.md model configuration section |
| T-03-03 | Tampering | artifacts array path traversal | accept | Low risk: artifacts are informational only (workflow uses them for logging, not for filesystem operations). Agent contract specifies paths must be under .planning/ or project root |
| T-03-04 | Denial of Service | Agent retry loop | mitigate | Hard limit of 1 retry per D-09. Documented in agent-contracts.md retry rules |
| T-03-05 | Information Disclosure | /tmp/ bridge file readable by other processes | accept | Low risk: bridge file contains only context usage percentage (no secrets). /tmp/ is standard OS temp directory. session_id validated against path traversal pattern |
| T-03-06 | Spoofing | Stale or missing /tmp/ bridge file | mitigate | Executor instructions specify: if file not found or timestamp older than 60 seconds, skip budget check for this cycle (fail-open, not fail-closed) |
</threat_model>
