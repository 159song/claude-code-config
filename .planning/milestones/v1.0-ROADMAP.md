# Milestone v1.0: WF Workflow System

**Status:** SHIPPED 2026-04-13
**Phases:** 1-6
**Total Plans:** 19

## Overview

WF 的优化路径从底层向上推进：先夯实 CLI 工具基础（每个后续改进都依赖它），再建立状态安全机制防止数据腐坏，然后规范 agent 交接合同实现可靠的自动化，接着补全会话管理让工作可中断可恢复，再增强工作流覆盖关键场景，最后补齐质量工具形成完整闭环。每个阶段交付一个独立可验证的能力提升。

## Phases

**Goal**: 工作流通过单次 CLI 调用获取所有所需上下文，wf-tools.cjs 从单文件进化为模块化架构
**Depends on**: Nothing (first phase)
**Requirements**: INFRA-01, INFRA-02, INFRA-03, INFRA-04, INFRA-05
**Plans**: 3 plans

Plans:
- [x] 01-01: Modularize wf-tools.cjs into router + 7 lib/ modules with findProjectRoot
- [x] 01-02: Implement compound init command with sub-modes
- [x] 01-03: Fix hook paths in settings.json and replace all {{WF_ROOT}} placeholders

### Phase 2: State Safety

**Goal**: 所有对 STATE.md 和 ROADMAP.md 的变更都通过 CLI 命令完成，消除直接文件修改造成的格式腐坏和并行冲突
**Depends on**: Phase 1
**Requirements**: STATE-01, STATE-02, STATE-03, STATE-04, STATE-05, STATE-06
**Plans**: 4 plans

Plans:
- [x] 02-01: Fix nested YAML parsing and implement frontmatter CRUD (get/set/merge/patch/validate)
- [x] 02-02: Create validate.cjs module for health check and auto-repair
- [x] 02-03: Implement phase transition commands and fix content-based verification
- [x] 02-04: Migrate workflow files from direct STATE.md writes to CLI commands

### Phase 3: Agent Contracts

**Goal**: 每个 agent 有明确的输入输出合同、完成标记和 context 预算规则，工作流可靠检测 agent 完成并正确交接
**Depends on**: Phase 2
**Requirements**: AGENT-01, AGENT-02, AGENT-03, AGENT-04
**Plans**: 2 plans

Plans:
- [x] 03-01: Add contracts, completion markers, and native API fields to 5 agent definitions + create agent-contracts.md reference
- [x] 03-02: Update 6 workflow files with contract-based invocation, completion parsing, retry logic, and config-driven models

### Phase 4: Session Management

**Goal**: 用户可以随时暂停工作、跨会话恢复上下文、并让系统自动推进到下一步，长任务不再被迫一次完成
**Depends on**: Phase 2, Phase 3

**Plans**: 3 plans

Plans:

- [x] 04-02: Create /wf-pause and /wf-resume commands with workflow, CLI integration, and settings.json update
- [x] 04-03: Create /wf-next auto-advance command with lifecycle detection and Skill() routing

### Phase 5: Workflow Enhancement

**Goal**: 工作流覆盖更多实际场景（自治执行、阶段动态调整、配置管理），减少卡顿和误报

### Phase 6: Quality Tools

**Goal**: 用户拥有代码审查工作流和里程碑管理能力，形成从编写到审查到发布的完整闭环
**Depends on**: Phase 5
**Requirements**: QUAL-01, QUAL-02
**Plans**: 3 plans

- [x] 06-03: Milestone lifecycle: complete-milestone and new-milestone commands and workflows

## Milestone Summary

**Key Decisions:**
- 6 phases derived from requirement categories with dependency ordering: CLI -> State -> Agent -> Session -> Workflow -> Quality
- Research confirms CLI foundation is highest-impact starting point (compound init eliminates 60-70% unnecessary context consumption)

**Issues Resolved:**
- Context 浪费问题通过 compound init 解决
- 状态腐坏通过 CLI-only writes 解决
- Agent 交接不可靠通过合同机制解决

**Technical Debt Incurred:**
- Phase 3-6 plans were marked complete in ROADMAP.md but phase artifacts were cleaned before formal archival
